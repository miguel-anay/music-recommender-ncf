Binarización (Feedback Implícito)

  • Se aplica log1p(weight) para suavizar la distribución
  • Umbral en la mediana → etiqueta 1 (like) o 0 (dislike)
  • Resultado: 35,732 likes y 35,679 dislikes (casi balanceado)

Negative Sampling

  • Por cada interacción positiva → 4 pares negativos aleatorios
  • Dataset final: 145,580 pares de entrenamiento
  • Garantiza que el modelo aprenda a distinguir preferencias
  • Sin data leakage verificado entre train y test ✓

Análisis de Resultados

¿Por qué SVD supera al NCF?
El SVD es un baseline muy sólido para datos densos. El NCF fue entrenado en CPU sin tuning extensivo y con solo 20 épocas.

Ventajas del NCF Híbrido
Incorpora tags semánticos (TF-IDF) que el SVD ignora. Puede modelar relaciones no lineales y está preparado para cold-start con contenido.

Protocolo de Evaluación
500 pares de test muestreados aleatoriamente, 100 candidatos negativos por positivo, mismo split 80/20 para ambos modelos.

Documentación y Presentación del Sistema

Reproducibilidad, trabajo futuro y conclusiones
📦

Repositorio reproducible en GitHub

Código modular en src/, notebooks ordenados (01→04), setup automático para Colab con un click. Cualquier persona puede ejecutar el proyecto desde cero.

🧠

Modelo híbrido: Collaborative Filtering + Content-Based

El NCF combina embeddings de usuarios y artistas con features semánticas de tags (TF-IDF 50d), logrando un sistema capaz de trabajar con información multimodal.

📊

Evaluación honesta con protocolo estándar

Hit Rate@10 = 0.70 y NDCG@10 = 0.41. El SVD baseline resulta competitivo, lo que motiva optimización del NCF (GPU, más épocas, tuning de hiperparámetros).

🚀
